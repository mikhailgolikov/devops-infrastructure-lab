# Лабораторная работа Высшая It-школа Томский Государственный Университет: Отказоустойчивая инфраструктура для приложения

## О проекте 
Развернута инфраструктура для приложения на Go с PostgreSQL и Redis-кластером. Для развертывания использована виртуалка дистрибутива Ubuntu на Linux. Использован Ansible, как способ автоматизации развертывания, а также Gitlab Ci.
Цель: Продемонстрировать навыки и опыт развертывания приложения, обеспечение ему отказоустойчивости и создание бэкапов базы данных.

## Архитектра 
- **ВМ:** Ubuntu 22.04
- **Приложение:** бэкенд на Go, запускается через systemd
- **СУБД:** PostgreSQL(на той же вм)
- **Кэш:** Redis Cluster (3 master + 3 replica) на одной ВМ
- **Бэкапы:** локальные ежедневные, синхронизация с Яндекс.Диск
- **Автоматизация:** Ansible 
- **CI/CD:** GitLab CI

## Технологии
- Linux 
- Systemd, logrotate
- PostgreSQL
- Redis (кластер)
- Ansible
- GitLab CI
- Bash(script)

## Ansible роли:
  - install_packages - установка всех необходимых пакетов на виртуальную машину
  - postgres_role  - установка Postgres
  - redis_role - настройка redis-кластера, настройка бэкапов
  - myapp -  выполнение типовых операций с  приложением: старт приложения, стоп приложения, рестарт приложения

### Gitlab CI(CI/CD)
Пайплайн запускается при пуше в ветку, выбранную основной, при изменение в коде Ansible или в коде приложения 

## Как происходит весь процесс развертывания
При пуше в репозиторий начинает выполняться пайплайн. Пайплайн запускает команду для запуска Ansible на моей ВМ(ansible-playbook -i go-app/AnsibleProject/inventory/host_vars.ini  go-app/AnsibleProject/playbook.yml)
